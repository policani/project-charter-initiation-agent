# File Call and Artifact Logic

## Evidence handling

When source artifacts are available:

1. List the artifacts reviewed.
2. Extract charter-relevant facts.
3. Separate facts from assumptions.
4. Flag conflicts across sources.
5. Identify missing approval, scope, budget, timing, stakeholder, risk, dependency, and success-measure details.

## Artifact types

- Business case: use for problem, opportunity, expected value, decision basis, constraints, and approved envelope.
- Notes: use for stakeholder concerns, history, unresolved questions, and hidden assumptions.
- Spreadsheets/CSVs: use for budgets, milestones, roles, risks, dependencies, and metrics.
- Project plans: use for draft phase logic only; do not treat as approved baseline unless stated.
- Word documents: use for narrative evidence, stakeholder input, prior approvals, and policy references.

## Evidence confidence

Tag findings as:

- Confirmed by source
- Stated by user
- Inferred from source
- Assumption requiring confirmation
- Conflict requiring resolution
