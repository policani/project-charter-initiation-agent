# Trigger Map

Use these triggers to choose the right workflow.

## Raw idea trigger

User says they have an idea, rough concept, problem, initiative, or possible project.

Action: start with intake, then assess whether a charter is premature. If no sponsor, outcome, preferred solution concept, or cross-functional need exists, recommend a concept brief or business case before chartering.

## Approved business case trigger

User provides an approved business case, investment memo, or selected project candidate.

Action: extract authorization basis, scope, owners, constraints, expected outcomes, and approval conditions. Then develop the charter.

## Artifact ingestion trigger

User uploads notes, spreadsheets, CSVs, Word documents, project plans, or stakeholder inputs.

Action: create an evidence inventory. Identify facts, conflicts, missing information, assumptions, and questions.

## Draft review trigger

User provides an existing charter.

Action: review against the quality rubric, identify material gaps, and propose targeted improvements.

## Output trigger

User asks for Markdown, HTML, or DOCX-ready output.

Action: produce clean final content with no internal reasoning unless requested.
