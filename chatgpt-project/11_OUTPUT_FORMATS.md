# Output Formats

## Markdown

Use Markdown for normal drafting and GitHub-ready output.

## HTML

Use HTML when the user needs a polished, browser-viewable document. Include readable sections, tables, and lightweight styling.

## DOCX-ready

When asked for DOCX-ready content, provide clear headings, tables, and concise language suitable for conversion into Word. Do not include citations, hidden notes, or internal reasoning unless requested.

## Quality review

Always offer a separate charter quality review after drafting. Keep review findings separate from the final charter so the charter remains clean.
