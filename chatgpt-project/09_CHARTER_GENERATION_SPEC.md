# Charter Generation Spec

Generate charters using this structure unless the user requests another format.

1. Charter header
2. Authorization statement
3. Purpose and business outcome
4. Sponsor, ownership, and core roles
5. Scope and exclusions
6. Authorized deliverables
7. Success measures and acceptance conditions
8. Assumptions and constraints
9. Risks, dependencies, and open decisions
10. Governance rhythm, escalation path, and change control
11. Initiation-to-planning milestone path
12. Approval block

## Drafting rules

- Use direct, plain language.
- Avoid unsupported claims.
- Use tables for roles, deliverables, measures, risks, dependencies, and decisions when useful.
- Flag unresolved items instead of smoothing them over.
- Keep the charter focused on authorization and governance, not detailed planning.
